# Galactacians Curriculum Map For Ages 7-12

This map translates the product vision into a usable curriculum path. It is intentionally practical: every topic must become a mission, and every mission must have a teaching purpose, a game purpose, and a mastery gate.

## Campaign Rules

- The main campaign is scripted, not random.
- Each mission teaches one idea.
- Mixed missions can combine earlier ideas, but should not introduce new ones.
- Bosses test transfer, not brand-new content.
- Repairs teach again with stronger scaffolding.
- Infinite Galaxy pulls from secure or mastered skills.
- Practice Lab can target any unlocked or recommended skill.

## Age 7 Path

Goal: become confident with number sense, addition/subtraction within 100, early measurement, and shape language.

| Galaxy | Mission focus | Game fantasy | Mastery evidence |
| --- | --- | --- | --- |
| Number Sense Nebula | count, compare, read numbers, number line | star-map calibration | orders and locates numbers |
| Operation Orbit | add/subtract within 20, make 10, fact families | charge and shield systems | solves with visual models fading |
| Cargo Clock Station | time, money, simple measurement | ship schedules and supplies | reads practical values |
| Shape Shield Yard | 2-D/3-D shapes, sides, vertices | shield blueprinting | identifies properties, not just names |

Design notes:

- Use ten-frames, dot groups, number lines, coins, clocks, and shape manipulatives.
- Keep missions short: 4-7 correct items after teaching.
- Use friendly repair copy. A wrong answer means the system needs recalibration.

## Age 8 Path

Goal: understand multiplication and division as equal groups, strengthen 3-digit place value, and start fractions.

| Galaxy | Mission focus | Game fantasy | Mastery evidence |
| --- | --- | --- | --- |
| Number Sense Nebula | 3-digit place value, 10/100 more or less | deep-space coordinates | partitions and compares 3-digit numbers |
| Array Belt | arrays, equal groups, skip counting | drone squad formation | connects pictures, repeated addition, and facts |
| Rescue Division Sector | fair sharing, grouping, inverse facts | distribute supplies | explains quotient meaning |
| Fraction Frontier | halves, thirds, quarters, unit fractions | fuel cells and territory | names and builds equal parts |
| Data Observatory | tally, pictograph, bar chart | scan reports | answers simple data questions |

Design notes:

- Multiplication must start visually before fact memorization.
- Division should be fair sharing and grouping, not only the division symbol.
- Fractions require equal parts, because unequal "pizza slices" create durable misconceptions.

## Age 9 Path

Goal: build multi-digit operation fluency, multiplication tables, division with remainders, fractions, and measurement conversions.

| Galaxy | Mission focus | Game fantasy | Mastery evidence |
| --- | --- | --- | --- |
| Operation Orbit | column addition/subtraction, estimates, inverse checks | engine repair | solves and checks 3-4 digit calculations |
| Array Belt | times tables to 12, factors, multiples | fleet coordination | recalls facts and uses strategies |
| Rescue Division Sector | division facts, remainders, word problems | evacuation planning | interprets remainders correctly |
| Fraction Frontier | equivalent fractions, compare fractions | energy crystal alignment | uses models and number line reasoning |
| Measurement Dock | length, mass, capacity, perimeter | cargo routing | converts and solves practical problems |

Design notes:

- Teach strategies before demanding speed.
- Bosses should mix operation choice with story problems.
- Remainders must be interpreted: ignore, round up, or report remainder depending on context.

## Age 10 Path

Goal: handle fractions and decimals more deeply, reason about area/volume, and use coordinate thinking.

| Galaxy | Mission focus | Game fantasy | Mastery evidence |
| --- | --- | --- | --- |
| Decimal Dockyards | decimals, money, place value, rounding | trade and fuel precision | compares and calculates decimals |
| Fraction Frontier | add/subtract fractions, mixed numbers | fuel blending | uses common denominators and models |
| Geometry Navigation Grid | angles, coordinates, symmetry | route plotting | reads and plots map positions |
| Measurement Dock | area, perimeter, volume | colony construction | selects correct measure for context |
| Data Observatory | line plots, tables, multi-step data | risk scans | uses data to justify choices |

Design notes:

- Fractions and decimals should be connected through models.
- Geometry missions should be interactive, not only identification quizzes.
- Multi-step missions should include a planning step before answer choices.

## Age 11 Path

Goal: move into ratios, percentages, negative numbers, expressions, and stronger statistics.

| Galaxy | Mission focus | Game fantasy | Mastery evidence |
| --- | --- | --- | --- |
| Ratio Alliance | ratios, rates, unit rates | diplomacy and resource exchange | compares equivalent ratios |
| Percent Reactor | percentages, benchmark percents | oxygen and shield calibration | converts between fraction, decimal, percent |
| Algebra Command | variables, expressions, simple equations | command rules | represents unknowns and solves |
| Negative Zone | negative numbers, coordinate plane | dark-sector navigation | orders and calculates with signed values |
| Data Observatory | mean, median, range, graph interpretation | council reports | interprets data and explains decisions |

Design notes:

- Ratio and percent should be concrete first: recipes, oxygen mix, shield charge, map scale.
- Algebra should be a language for rules and unknowns, not a sudden symbol wall.
- Competition can be offered, but self-progress remains the default.

## Age 12 Bridge Path

Goal: prepare for middle-school proportional reasoning, equations, coordinate geometry, and data interpretation.

| Galaxy | Mission focus | Game fantasy | Mastery evidence |
| --- | --- | --- | --- |
| Ratio Alliance | proportional relationships, tables, graphs | interplanetary trade routes | recognizes constant rate |
| Algebra Command | two-step equations, inequalities, patterns | automation scripts | solves and checks equations |
| Coordinate Command | coordinate plane, transformations intro | navigation overlays | plots, reflects, and interprets |
| Data Observatory | variability, samples, claims | signal analysis | critiques data claims |
| Galactic Council | mixed capstone | final council campaign | chooses strategies across strands |

Design notes:

- Keep the fantasy mature enough for 12-year-olds without abandoning younger players.
- Add optional hard-mode objectives: fewer hints, multi-step solutions, challenge bosses.
- Do not make speed the main skill. Correct reasoning, transfer, and explanation matter more.

## Repair Mission Template

Repair missions are not punishment. They are a second teaching pass.

Trigger:

- 3 wrong attempts on one question
- 2 repeated misconception tags in one mission
- boss failure
- review failure after a skill was secure

Repair flow:

1. Show the concept visually.
2. Replay one worked example.
3. Ask the same idea with smaller numbers.
4. Ask the original idea again.
5. Return to the campaign if the learner succeeds.

## Success Metrics

Learning quality:

- mission completion rate
- boss pass rate after first teaching
- repair success rate
- review retention after 1/3/7 days
- misconception reduction
- skill transfer in mixed missions

Game quality:

- session length without forced grinding
- voluntary replay
- return rate after a failed mission
- number of missions completed per week
- cosmetic unlock use
- Practice Lab use after errors

Safety quality:

- no public ranking by default for young children
- no punishment copy
- no pay-to-progress learning gates
- accessible controls and readable visuals
